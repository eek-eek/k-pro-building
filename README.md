# AI Smeta KZ

Веб-приложение для подготовки **базовой (ориентировочной) сметы здания** по
строительным нормам Республики Казахстан. Заказчик вводит понятные параметры
объекта — система рассчитывает объёмы материалов и работ и формирует смету с
разбивкой по разделам, накладным, резерву и НДС.

Ключевая идея: **нормативные знания (ГОСТ/СНиП/СН РК) один раз разбираются LLM и
складываются в БД как структурированные правила**. Повторные расчёты того же
профиля объекта берут готовые правила из БД и не тратят токены на LLM.

См. [ARCHITECTURE.md](ARCHITECTURE.md) — роли, системы и поток данных.

## Стек

- **Backend:** Python 3.11 + FastAPI, SQLAlchemy (SQLite по умолчанию,
  Postgres-ready), SSE для статусов.
- **LLM:** провайдер-абстракция (Gemini по умолчанию; Anthropic / OpenAI / demo).
- **Frontend:** статический HTML/CSS/JS (на базе исходного MVP), общается с бэком
  по REST + Server-Sent Events.

## Быстрый старт

```bash
cp .env.example backend/.env      # заполните GEMINI_API_KEY (или другой провайдер)
./run.sh                          # поднимет http://127.0.0.1:8000
```

Откройте <http://127.0.0.1:8000> — это и фронтенд, и API.

Без ключа LLM приложение работает в режиме дефолтных нормативных коэффициентов РК
(укрупнённые показатели). Поставьте галочку «Демо без LLM» для явного оффлайн-расчёта.

## Развёртывание на сервере

Готовый деплой на VPS (AWS EC2, Ubuntu) — systemd + Caddy (логин/пароль, позже HTTPS):
см. [`deploy/DEPLOY.md`](deploy/DEPLOY.md). Кратко: `git clone` → `bash deploy/setup.sh`
→ настроить Caddy. Обновление — `bash deploy/update.sh`.

## Поток

1. Пользователь вводит атрибуты здания → фронт `POST /api/estimate` → `job_id`.
2. Бэк в фоне: разбор атрибутов → резолв норм (кэш БД → правила → LLM → дефолты)
   → расчёт объёмов → формирование сметы.
3. Фронт слушает `GET /api/estimate/{job_id}/events` (SSE) и показывает шаги.
4. По готовности бэк отдаёт JSON-смету; фронт рендерит таблицы и даёт экспорт TXT.

## API

| Метод | Путь | Назначение |
|------|------|-----------|
| POST | `/api/estimate` | создать задачу расчёта (async), вернуть `job_id` |
| GET  | `/api/estimate/{job_id}` | статус задачи (поллинг) |
| GET  | `/api/estimate/{job_id}/events` | поток статусов (SSE) |
| POST | `/api/estimate/sync` | синхронный расчёт (без статусов) |
| GET  | `/api/estimates/{id}` | сохранённый расчёт |
| GET  | `/api/norms` | реестр нормативных документов |
| GET  | `/api/prices` | справочник цен |
| GET  | `/api/health` | проверка |

## Тесты

```bash
cd backend && python -m pytest -q
```

## Конфигурация

Все настройки — в `backend/.env` (см. `.env.example`): провайдер и ключи LLM,
модели, web-grounding, строка подключения к БД, TTL кэша знаний, CORS.

## Дисклеймер

Смета предварительная (класс точности 5), цены индикативные и требуют уточнения.
Не является основанием для договоров подряда. Неподтверждённые нормы помечаются
«требует проверки сметчиком» — номера документов не выдумываются.
