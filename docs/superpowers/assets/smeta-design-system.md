# Смета Design System v1.0 — референс для фронтенда (План 3)

Источник: PDF «Дизайн-система сметы» (v1.0 · 27.06.2026), присланный пользователем.
Назначение: единый язык интерфейса для рабочего места сметчика. Приоритеты —
данные на первом месте (моноширинные числа, ровные колонки, выравнивание по
разряду), спокойный интерфейс (тёплый фон, один акцент, цвет только для статусов),
плотность под профи (компактные строки с воздухом).

**Адаптация под проект (KZ):** визуальную систему берём как есть; валюту и нормы
адаптируем — **₸** вместо ₽, **СН РК / ГОСТ** вместо ФЕР-2020. Числа форматируем с
пробелами-разделителями (`ru-RU`).

## Токены (CSS-переменные)

```css
:root {
  /* поверхности */
  --bg: #F4F3EF;          /* тёплый нейтральный фон страницы */
  --surface: #FFFFFF;     /* карточки, таблицы, поля */
  --text: #1C2024;        /* основной текст */
  --border: #E7E5DF;      /* hairline-границы, разделители */

  /* акцент */
  --accent: #2C5BA8;      /* primary (синий) */
  --accent-tint: #E8EEF7; /* светлый фон (tint-кнопка, подсветка строки раздела) — приближённо */
  --accent-mid: #6F93C9;  /* средний оттенок — приближённо */
  --accent-dark: #1E3F73; /* тёмный оттенок (hover/press) — приближённо */

  /* семантика статусов */
  --status-approved: #2F7A55; /* Утверждена */
  --status-review:   #9A6A12; /* На согласовании */
  --status-rejected: #BB433B; /* Отклонена */
  --status-draft:    #949CA6; /* Черновик */

  /* типографика */
  --font-ui: "Golos Text", system-ui, -apple-system, sans-serif;
  --font-mono: "JetBrains Mono", ui-monospace, "SF Mono", monospace;
}
```

Оттенки акцента `--accent-tint/-mid/-dark` в PDF даны только образцами без hex —
значения приближённые, при реализации можно уточнить пипеткой по PDF.

## Типографика

- Семейства: **Golos Text** — интерфейс и заголовки; **JetBrains Mono** — все
  числа, суммы, коды, номера (моноширинный → колонки выравниваются по разряду).
- Шкала:
  - **H1** — 44px / вес 700 (заголовок страницы)
  - **H2** — 22px / 700 (заголовок раздела)
  - **Body** — 14px / 400 (текст, подписи полей, описания позиций)
  - **Caption** — 12px / 600, UPPERCASE, цвет приглушённый (метки, ЗАГОЛОВКИ КОЛОНОК)
- Доступные веса Golos: 400 500 600 700 800.
- Подключение шрифтов: веб-фонты (Golos Text и JetBrains Mono есть на Google Fonts).

## Компоненты

### Кнопки (размеры Small / Medium / Large)
- **Primary** — заливка `--accent`, белый текст («Создать смету», «+ Позиция»,
  «+ Новая смета»).
- **Secondary** — `--surface` фон, `--border` рамка, тёмный текст («Дублировать»,
  «Экспорт PDF» / у нас «Экспорт Word», «Фильтры»).
- **Tint** — фон `--accent-tint`, текст `--accent` («+ Позиция» / «+ ресурс»).
- **Danger** — текст `--status-rejected`, без заливки («Удалить»).

### Поля ввода
- Текстовое (наименование), число (выравнивание **вправо**, моно), select с «▾»
  (единицы «м² ▾»), поиск с иконкой «⌕» (placeholder «Поиск по справочнику…»).
- Подпись поля — Caption сверху; рамка `--border`, фон `--surface`.

### Статусы и теги
- **Бейдж статуса**: точка ● + подпись, цвет по семантике
  (Утверждена/На согласовании/Отклонена/Черновик). Фон — светлый тон цвета.
- **Тег**: пилюля с рамкой (`СН РК 8.02-05`, `Объект: офис`); код/`НДС 12%` — моно.

### Таблица сметы (главный компонент)
Колонки: `№ · НАИМЕНОВАНИЕ · ЕД. · КОЛ-ВО · ЦЕНА · СУММА`.
- Заголовки колонок — Caption UPPERCASE, приглушённые.
- **Строка-раздел**: синий номер раздела, жирное название слева, **сумма раздела**
  справа; фон строки — лёгкая подсветка (`--accent-tint` слабая).
- **Строка-позиция**: номер `01/02…` моно серый; наименование; ед. серый;
  кол-во/цена/сумма — моно, **вправо**; цена приглушённая, **сумма жирная тёмная**.
- Разделители строк — hairline `--border`.

### Блок итогов
Строки: `Работы · Материалы · Итого без НДС · НДС % ·` → последняя строка
**«Всего к оплате»** в заливке `--accent` (белый текст, крупная моно-сумма).
Наш аналог: Прямые / Накладные / Резерв / НДС / **ИТОГО с НДС**.

## Экраны

### Редактор сметы (= наш экран детали)
- Шапка: H2-название объекта + бейдж статуса; подзаголовок моно
  («№ … · заказчик · ред. дата»); справа кнопки Secondary («Экспорт») + Primary
  («+ Позиция»).
- Таблица с группами-разделами и позициями (см. выше).
- Блок итогов справа/снизу.
- **Сюда же ложится ресурсная детализация v0.3:** раскрытие строки-позиции →
  вложенные строки-ресурсы (материал/труд/машины) с моно-числами; «+ ресурс»
  (Tint-кнопка), «Удалить» (Danger) на ресурсе.

### Список смет (= наш дашборд)
- Шапка: поиск ⌕ + Secondary «Фильтры» + Primary «+ Новая смета».
- Таблица: `НОМЕР (моно, синий-ссылка) · ОБЪЕКТ (жирн.) · ЗАКАЗЧИК (серый) ·
  СУММА (моно вправо) · СТАТУС (бейдж)`.

## Принцип
Все экраны — на одних токенах и компонентах. Акцент и плотность строк
переключаемы (в PDF — панель Tweaks); для нас достаточно зафиксировать значения
токенов в `:root`, переключатель плотности — опционально (вне области v0.3).
